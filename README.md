# 📊 Модель данных Data Vault с партиционированием

Предоставленный SQL-скрипт реализует физическую схему базы данных по методологии **Data Vault 2.0** в СУБД PostgreSQL. Схема включает хабы, линки, сателлиты, механизмы партиционирования и тестовые запросы.

---

## 🏛️ Хабы (Hubs)
Хранят уникальные бизнес-ключи сущностей.

*   `Customers_hub` — Клиенты (`customer_id`)
*   `Products_hub` — Товары (`product_id`)
*   `Orders_hub` — Заказы (`order_id`)
*   `Order_Statuses_hub` — Статусы заказов (`status_id`)

---

## 🔗 Ссылки (Links)
Отображают связи M2M (многие-ко-многим) между бизнес-ключами.

*   `Orders_Products_link` — Связь заказа и товаров
*   `Orders_Customers_link` — Связь заказа и клиента
*   `Orders_Statuses_link` — Связь заказа и его статуса

---

## 🛰️ Сателлиты (Satellites)
Хранят описательные данные и историю изменений.

*   `Customers_sat` — Данные клиента (имя, email, телефон)
*   `Products_sat` — Характеристики товаров (название, категория)
*   `Order_Statuses_sat` — Текстовое описание статуса
*   `Orders_sat` — Параметры заказа (сумма, количество). **Партиционирована**

---

## ✂️ Партиционирование таблицы `Orders_sat`
Таблица `Orders_sat` физически разделена на основе диапазонов дат колонки `load_date`:

*   `orders_2024` — Данные с `2024-01-01` по `2024-12-31`
*   `orders_2025` — Данные с `2025-01-01` по `2025-12-31`

📌 Для ускорения поиска по партициям созданы индексы `idx_order_hash_key` и `idx_load_date`.
